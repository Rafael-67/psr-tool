# Validation checklist

## Functional regression

- [ ] `index.html` opens all three application modules.
- [ ] `tool.html` completes Steps 1–8 in English and Spanish.
- [ ] Structured TXT/PDF print report is generated.
- [ ] `platform.html` creates, edits, duplicates, stores and deletes assessments.
- [ ] Existing `psr:index`, `psr:profile`, `psr:lang` and `psr:<id>` data remain readable.
- [ ] `analytics.html` reads platform assessments and imported CSV.
- [ ] Cohen's κ and Fleiss' κ are calculated without errors.

## Scientific conformity with the published article and Supplementary S1

- [ ] Agent characterization includes authoritative RG source and GMO characteristics.
- [ ] Every procedure records volume, concentration, aerosol potential, equipment and frequency.
- [ ] Highest-risk procedural step is identifiable.
- [ ] PSR is classified as Low, Moderate or High.
- [ ] Nine procedural modulating factors are available.
- [ ] Individual susceptibility is documented separately and excluded from the ≥2 escalation count.
- [ ] ≥2 unfavorable factors trigger escalation.
- [ ] Likelihood × severity matrix is completed before BSL assignment.
- [ ] BSL assignment includes written technical justification.
- [ ] Engineering, PPE and administrative controls are documented.
- [ ] Reassessment triggers and next review are documented.
- [ ] Methodology verification checklist is complete before institutional review.

## Worked-example regression

Validate the six published scenarios against Supplementary Appendix S2, including escalation, reduction and dual-technology comparisons.
